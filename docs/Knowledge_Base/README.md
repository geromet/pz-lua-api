# Knowledge Base

Stable reference documents about how this project works, why it works that way, and how to work on it. These files change slowly and should reflect the current reality of the codebase.

**When to read this folder:** When you need to understand the domain, an architectural decision, or the project's style conventions.
**When to write here:** When a significant decision is made (add an ADR to `Decisions.md`), when new domain terminology is introduced (add to `Domain-Language.md`), or when a coding pattern is established (add to `Design-Patterns.md`).

## Files

| File | Contents |
|------|---------|
| `Decisions.md` | Architectural Decision Records (ADR-001 and onwards) |
| `Domain-Language.md` | Project-specific terminology and what things mean |
| `Design-Patterns.md` | Recurring code patterns used in this codebase |
| `Style-Guide.md` | Naming, formatting, and code style conventions |
| `Philosophy.md` | What this project is for and what it is not |
| `Working-Agreement.md` | How Claude and the user collaborate on this project |
| `Testing.md` | How to test changes (no automated tests — manual verification steps) |
| `Refactoring.md` | Notes on planned or in-progress structural changes |
